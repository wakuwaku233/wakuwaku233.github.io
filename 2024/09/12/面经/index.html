2. 前端数据存储方式及其应用场景

  • Cookies: 用于存储少量数据，适合需要与服务器交互的场景，例如保存登录状态。受限于大小（一般不超过 4KB）和安全性（易被篡改）。
  • LocalStorage: 存储数据在客户端，数据持久化，适合存储较大数据（5MB），例如用户偏好设置。数据不会过期，除非手动删除。
  • SessionStorage: 存储数据在客户端，数据只在当前会话中有效，适合临时数据存储，如表单输入的暂存。数据在浏览器关闭时清除。
  • IndexedDB: 浏览器内置的 NoSQL 数据库，用于存储大量结构化数据，适合离线应用和需要复杂查询的场景。
  • WebSQL: 已被废弃的浏览器 API，用于存储结构化数据。建议使用 IndexedDB 作为替代方案。

3. 受控组件和非受控组件

受控组件：

  • 定义: 组件的状态由 React 的状态（state）或其他状态管理工具完全控制。

  • 特征: 表单元素的值由 React 状态决定，每次输入变化都会更新状态。

  • 优点: 易于管理和验证数据，便于实现复杂的交互逻辑。

非受控组件：

  • 定义: 组件的状态由 DOM 节点自身控制，React 状态不会直接干预。
  • 特征: 使用 ref 访问 DOM 元素，数据的变化不会直接同步到 React 状态。
  • 优点: 简化了组件逻辑，适合对表单数据的简单处理。

4. 虚拟DOM

虚拟dom是一种优化技术，用于提高前端框架（如 React）的性能。其核心概念是：

  • 虚拟 DOM: 是对实际 DOM 的轻量级表示。每当组件状态改变时，框架会创建一个新的虚拟 DOM 树。
  • 对比和更新: 虚拟 DOM 通过与当前虚拟 DOM 树进行比较（称为“diffing”），找出变化部分，然后只更新实际 DOM 中的这些部分。
  • 效率: 这种方式减少了对实际 DOM 的操作次数，提高了应用的性能和响应速度，因为直接操作实际 DOM 是昂贵的操作。

例如，在 React 中，当状态改变时，虚拟 DOM 机制会计算出最小的更新量，优化了页面的渲染效率。

5.Javascript 数据溢出的控制方法

  • 使用大数库: 如 BigInt 或第三方库（例如 bignumber.js）来处理超大或超小的数值。
  • 数据验证: 在处理用户输入或计算结果之前，进行范围检查，确保值在预期范围内。
  • 防止精度丢失: 对浮点数运算进行适当处理，避免精度丢失。使用 toFixed、Math.round 等方法控制小数点后位数。
  • 安全整数检查: 使用 Number.isSafeInteger() 确保数值在安全整数范围内。

6.前端的网络安全 ？（XSS CSRF）

跨站脚本攻击（XSS）

定义: XSS（Cross-Site Scripting）是一种攻击方式，攻击者将恶意脚本注入到网页中，当其他用户访问该页面时，恶意脚本会在其浏览器中执行，从而窃取用户信息或执行其他恶意操作。

防护措施:

  • 输出编码: 对用户输入进行正确的编码，确保在插入到 HTML、JavaScript、CSS 等内容中时不会被解释为脚本。可以使用函数如 encodeURIComponent()、escape()、htmlspecialchars() 等。

  • 输入验证: 验证和过滤用户输入，确保数据符合预期格式，避免恶意代码注入。

  • 使用 Content Security Policy (CSP)

    : CSP 是一种安全机制，通过设置Content-Security-Policy头，限制网页可以加载的资源和脚本来源，从而减少 XSS 攻击的风险。

  • 避免 eval 和动态代码执行: 避免使用 eval()、setTimeout() 和 setInterval() 等可能执行动态代码的功能。

跨站请求伪造（CSRF）

定义: CSRF（Cross-Site Request Forgery）是一种攻击方式，攻击者诱使用户在已认证的网页上执行不希望的操作，例如提交表单或发起请求，从而劫持用户的会话。

防护措施:

  • 使用 CSRF 令牌: 在每个敏感操作中，要求附带 CSRF 令牌。令牌应随机生成且与用户会话关联，防止被伪造。

  • 使用 SameSite Cookie 属性: 将 Cookie 的 SameSite 属性设置为Strict或Lax，防止第三方站点发起的跨站请求带上 Cookie。

9.闭包是什么？

闭包是一种编程概念，指的是函数可以访问其外部函数作用域的变量。简单来说，它允许一个函数记住并访问其定义时的作用域，即使在外部函数已经执行完毕后。闭包常用于创建私有变量和函数，提供数据封装和保护。

适用场景：

  1. 数据私有化：
    • 通过闭包，可以创建私有变量，防止外部直接访问和修改。例如，使用闭包封装状态。
  2. 函数工厂：
    • 通过闭包创建特定行为的函数。例如，生成带有特定参数的函数。
  3. 事件处理器：
    • 在事件处理程序中保持对外部变量的引用，常用于回调函数和定时器。
  4. 记忆化函数：
    • 使用闭包存储计算结果，以提高性能。适用于一些重复计算的函数。
  5. 模块模式：
    • 通过闭包封装模块的私有部分和公共接口，组织代码结构。
  6. 延迟执行：
    • 在异步操作中保存上下文状态，便于在后续操作中访问。

10. 闭包和原型链的关系是什么？

闭包和原型链的关系

  • 独立性: 闭包和原型链是两个独立的概念。闭包与函数的作用域有关，而原型链与对象的继承有关。
  • 作用域 vs. 继承:
    • 闭包: 主要用于访问和维护函数外部的变量，不涉及对象继承。
    • 原型链: 主要用于对象的继承和方法共享，不涉及函数的作用域。
  • 相互独立的使用场景:
    • 闭包: 适用于需要封装数据或函数的场景，例如创建私有变量和方法。
    • 原型链: 适用于需要对象继承和共享方法的场景，例如构建继承体系和实现继承。

11. http1，1.1，2都有哪些区别？

  • HTTP/1.0：每次请求需要建立新连接，性能较低。
  • HTTP/1.1：引入持久连接和管道化，减少了连接开销，但仍有队头阻塞问题。
  • HTTP/2：通过多路复用、头部压缩和流量控制显著提升了性能和效率，解决了HTTP/1.x中的许多问题。

这些改进使得 HTTP/2 能够在现代网络应用中提供更高效的传输性能。

12. 首部压缩是什么？

首部压缩是指在HTTP/2协议中，对请求和响应头信息进行压缩的技术。HTTP/2使用一种称为HPACK的算法来压缩头部信息，减少传输的数据量。这种压缩方法通过使用静态表和动态表来存储和重用常见的头部字段，从而减少重复数据和降低网络带宽消耗，提高传输效率。

13. 二进制帧的好处是什么？

二进制帧在HTTP/2中提供了多个好处。首先，它提高了传输效率，因为二进制格式比文本格式更紧凑和解析速度更快。其次，它允许多路复用，意味着多个请求和响应可以在一个连接上并发传输，而不会互相阻塞。最后，二进制帧使得协议解析更加清晰和准确，减少了文本协议中的歧义问题，从而提高了整体的可靠性和性能。

15. 盒子类型

标准盒子模型：元素的宽度和高度只包括内容区域，不包括padding、border和margin。计算宽度时需手动加上内边距和边框的高度。

替代盒子模型：元素的宽度和高度包括content、padding和border。这样可以更方便地控制元素的尺寸，避免了手动计算的复杂性。可以通过设置 box-sizing: border-box; 来启用。

16. http 和 https 的区别

HTTP（超文本传输协议）和 HTTPS（安全超文本传输协议）之间的主要区别在于安全性。

  1. 安全性：
    • HTTP：数据以明文形式传输，容易受到中间人攻击，数据可能被窃取或篡改。
    • HTTPS：在 HTTP 的基础上，使用 SSL/TLS 加密协议，确保数据在传输过程中的安全性和完整性。
  2. 端口：
    • HTTP：通常使用端口 80。
    • HTTPS：通常使用端口 443。
  3. 证书：
    • HTTP：不需要安全证书。
    • HTTPS：需要 SSL/TLS 证书，以验证服务器身份并加密通信。

因此，HTTPS 更适合处理敏感信息，如在线支付和个人数据。

17. vue的生命周期

  1. 创建阶段：
    • beforeCreate：实例被创建，数据观测和事件配置尚未开始。
    • created：实例创建完成，数据观测和事件配置已完成，可以访问数据。
  2. 挂载阶段：
    • beforeMount：在挂载开始之前被调用。
    • mounted：实例被挂载后调用，此时可以操作 DOM。
  3. 更新阶段：
    • beforeUpdate：数据变化后，虚拟 DOM 重新渲染之前调用。
    • updated：虚拟 DOM 重新渲染并渲染后调用。
  4. 销毁阶段：
    • beforeDestroy：实例销毁之前调用，可以进行清理。
    • destroyed：实例销毁后调用。

18. scoped原理

  1. 动态生成类名：Vue 在编译组件时，为每个 scoped 样式生成一个唯一的类名，比如 data-v-xxxxxx，并将其应用到组件的根元素上。
  2. 选择器自动加前缀：所有 scoped 样式中的选择器自动加上这个唯一的类名前缀，确保样式仅影响该组件的元素。
  3. 样式隔离：通过这种方式，组件的样式不会影响其他组件，从而实现样式的局部化和隔离。

1. vue3新特性有哪些？

  • Composition API

    提供了一种新的组织组件逻辑的方法，使代码更加模块化和复用性更高。通过 setup 函数，可以在一个组件内组合和重用状态逻辑。

  • Reactive API

    改进了响应式系统，支持 ref 和 reactive 来创建响应式数据。响应式系统变得更加高效和灵活。

  • Teleport

    允许将子组件的 DOM 节点传送到 DOM 树的任意位置，方便在全局范围内管理和操作组件。

  • Fragments

    Vue 3 允许组件返回多个根节点，避免了 Vue 2 中必须使用单一根节点的限制。

  • Suspense

    用于处理异步组件的加载和等待状态，使得异步数据的处理变得更加优雅。

  • Improved TypeScript Support

    Vue 3 对 TypeScript 的支持更加全面和友好，增强了类型推导和类型安全性。

  • Performance Improvements

    Vue 3 提供了更快的虚拟 DOM 和响应式系统，减少了渲染和更新的开销，提高了整体性能。

  • Custom Renderer API

    允许开发者创建自定义渲染器，以便将 Vue 用于非 DOM 平台（例如原生应用、桌面应用等）。

  • Composition API with TypeScript

    改进了 Composition API 的 TypeScript 支持，使得类型推导和自动补全更加准确。
